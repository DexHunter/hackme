pesudo-ref problem in git

I was trying to deploy this blog from my gh-pages branch. But when I tried to add a worktree according to official guide I met fatal: Refusing to point HEAD outside of refs/. I have no idea what went wrong at the beginning, and checked official git documents. But still no clue, then I found this post claming the problem was caused by out-dated git. I then updated the git with additional ppa, the problem was finally solved.

Besides, I found out that when doing first-time add remote by git remote add origin <url>, remotes/origin/HEAD is unassigned. I need to do git remote set-head origin <branch> thanks to this post

Finally, there are some other ways which make deployment eaiser. Check this out. One of my friends suggests to use netlify as well.

Cheers!
